03 — Technical Architecture

LangGraph State Machine

# BookingState — the single source of truth class BookingState(TypedDict): messages: Annotated[list, add_messages] user_name: str pickup_zone: str dest_zone: str fare_kes: int pesapal_url: str current_step: str # Node check - fuzzy matched geofences def validate_pickup_node(state): zone = find_zone(state["pickup_raw"]) if zone: return {"pickup_zone": zone, "step": "destination"} return {"step": "collect_pickup"}

Key design decisions:

  • Dispatcher pattern — one node reads current_step and routes, keeping the graph simple
  • Fuzzy Matcher — difflib-based matching filters user input against the 19 zones
  • add_messages reducer — conversation history is append-only and never lost
  • FAQ intercept — routes to RAG mid-booking, then resumes seamlessly
  • Human-in-the-loop — no booking is confirmed without explicit user confirmation

Why RAG over pure LLM?

A raw LLM might invent pricing (e.g., "KES 50 base fare") or service areas that don't exist. RAG constrains the model to answer only from verified company policy — eliminating hallucination in FAQ responses.

Test query results:

Only name, pickup, destination collected. No phone, ID, or card details stored.

09 — Reflection

What Worked, Challenges & Next Steps

✅ What Worked Well

  • LangGraph dispatcher pattern — clean, auditable, and easy to extend
  • RAG grounding eliminated hallucinated pricing and service areas
  • PesaPal delegated payments kept card credentials out of database
  • Streamlit custom CSS isolation allowed native look and feel

⚠️ Challenges Encountered

  • Streamlit iframe constraints required dynamic height estimation
  • Live PesaPal REST authentication and signature checking
  • Fuzzy matching user entries without blocking natural sentences
  • Local testing constraints mitigated by polling status checks

💡 Lessons Learned

  • Start with a modular core layout before introducing third-party APIs
  • KDPA compliance is simpler when data minimization is built-in
  • Fuzzy match thresholds need padding to prevent false positives
  • Decoupling the billing stack secures the architecture

🚀 Future Iterations

  • Google Maps routing API integration for precise driving lines
  • Swahili language localization using multilingual models
  • NER name parsing to replace simple heuristic string splitting
  • Independent KDPA Data Protection Impact Assessment (DPIA)
